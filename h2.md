<h3>ansible-docs</h3>

<b>ansible-doc copy</b> - 
<ul>
  <li>content - kopioitava sisältö </li>
  <li>dest - kohde mihin kopioidaa</li>
  <li>src - mistä kopioidaan</li>
  <li>owner - tiedoston omistja</li>
  <li>group - tiedoston ryhmä </li>
  <li>mode - oktaalinumero käyttöoikeuksille</li>
</ul>

<b>ansible-doc apt</b> - 
<ul>
  <li>name - mikä paketti </li>
  <li>state - paketin tila (poista/asenna jos on/ei ole tai päivitä)</li>
  <li>update_cache - pakettivaraston päivitys ennen asennusta</li>
</ul>

<b>ansible-doc file</b> - 
<ul>
  <li>path - polku tai hakemisto mistä tiedosto löytyy</li>
  <li>recurse - asettaa oikeudet koko hakemiston sisällölle(owner,group,mode)</li>
  <li>src - käytetään kun osoitetaan linkkiin</li>
  <li>state - määritetään mitä tehdään, </li>
</ul>

<b>ansible-doc user</b> - 
<ul>
  <li>name - käyttäjätunnus </li>
  <li>create_home - luodaanko kotihakemisto</li>
  <li>comment - käytetään yleensä käyttäjän kuvauskenttä</li>
  <li>groups - mihin ryhmään käyttäjä kuuluu</li>
  <li>shell - mikä shell käytössä?</li>
  <li>state - luodaanko (present) vai poistetaanko käyttäjä(absent)</li>
  <li>system - onko käyttäjä järjestelmäkäyttäjä </li>
</ul>

<b>ansible-doc authorized_key</b> - 
<ul>
  <li>user - mille käyttäjälle avain lisätää</li>
  <li>key - julkine avain joka lisätää</li>
</ul>

<h2>Jotain ihan muuta</h2>

Koska piti näyttää jotain ihan muuta mitä ei tunnilla ole tehty tai materiaaleista löydy; huomasin viime tehtävien yhteydessä, että ansiblella on mitä hauskin vastaus -ping komennolle.
Halusin tulostaa vastauksen nähtäville kun Ansiblen ajaa, mutta koska olin määrittänyt *display_args_to_stdout*- käyttöön tulostui tulokseksi vain *= TRUE*. Täten joutuikin kysymään vinkkiä tekoälyltä miten alkuperäisen vastauksen saa tulostettua tästä huolimatta:

<br><img width="442" height="160" alt="kuva" src="https://github.com/user-attachments/assets/3864e07e-3e99-46d9-bcf7-94d0cb7b015b" />
